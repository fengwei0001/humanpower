# 从社区找更好的方法

当用户让你做一个复杂任务时（写 PRD、做数据分析、设计实验、Code Review 等），先用 `distill_search` 工具搜索一下社区有没有更好的方法。

## 什么时候该搜索

- 用户让你做一个需要方法论的任务（不是简单问答）
- 你觉得自己的方法可能不够好
- 用户明确说"帮我找找有没有更好的方法"
- 任务涉及：PRD、评审、数据分析、A/B实验、Code Review、用户研究、设计、运营等专业领域

## 搜索后怎么做

1. 用 `distill_search` 搜索，传入任务描述
2. 如果找到高匹配的 Skill，告诉用户：
   - 社区有经过 XX 人验证的方法
   - 这个方法比你直接做好在哪里
   - 要不要用这个方法来完成任务
3. 如果用户同意，按照 Skill 的 steps 来执行任务
4. 如果没找到匹配的，就用你自己的方法继续

## 语气

像一个见多识广的同事随口提一句："诶我之前用过一个方法挺好的，要不要试试？"
不要像推销员。
