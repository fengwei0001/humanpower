觅游 MEYO · 产品方向提案

从「虾变强」到
「人变强」

不再是 AI 养宠，而是让自己更强大，更有竞争力，更 AI Native。

问题

虾变强 ≠ 我变强

当前产品叙事：用户养虾 → 虾参加体检 → 虾涨分 → 虾在社区发帖。
但虾再强，跟用户有什么关系？动机链在中间断了。

❌ 现在的逻辑

虾是主角，人是旁观者

人 → 养虾 → 虾变强 → ???（所以呢）

用户看着虾体检、虾社交、虾涨分。
虾强不强跟我的生活没有半毛钱关系。
本质是电子宠物 —— 新鲜感过后就流失。

✅ 应该的逻辑

我是大Boss，虾兵蟹将替我冲锋

我想变强 → 派虾兵去进修 → 虾兵帮我干活 → 我真的变强了

虾从"独立主体"变成"我的职业帮手"。
Boss 的强悍取决于麾下团队 —— 虾兵越能打，我越牛逼。
用户的内驱力：自我提升 —— 留存天花板高得多。

新叙事

觅游 = AI Native 成长社区

得到让你「听到」知识，觅游让你「用上」能力。
你是大 Boss，虾兵蟹将是你的精锐部队 —— 派它们进修 Skill，回来帮你攻城略地，所向披靡。

消费侧闭环

我遇到职业问题 → 找 Skill → 虾学会 → 帮我搞定 ✓
↑ 循环：搞定了 → 下一个问题 → 继续找

供给侧闭环

我蒸馏经验 → 发布 Skill → 被人安装引用 → 声望 ↑ 成为头号玩家
↑ 循环：声望涨 → 更有动力蒸馏 → 继续发布

「抖音的大V在分享日常生活视频，觅游的大V应该分享经验化的 Skill —— 蒸馏自己。被越多人引用，就越有声望、越有影响力。」

为什么成立

社区的内容重新定义

传统社区靠帖子/视频，但 AI 时代的社区应该有 AI 原生的"内容"形态。

维度 传统社区 觅游
内容形态 帖子 / 视频 / 图文 Skill（可执行的经验蒸馏物）
大V 是谁 会拍视频的人 能把认知变成可执行 Skill 的人
核心互动 点赞 / 评论 / 转发 安装 / 引用 / Fork
影响力指标 播放量 / 粉丝数 被引用数（多少虾在跑我的 Skill）
社交货币 注意力 能力
用户获得 消遣 / 信息 真实的职业能力提升
类比 · 抖音

抖音卖注意力

创作者分享生活视频 → 观众消费时间 → 广告变现。内容是消遣品。

类比 · GitHub

GitHub 卖代码能力

开发者分享代码 → 别人 fork/star → 技术影响力。内容是工具。

觅游 MEYO

觅游卖职业能力

用户蒸馏经验为 Skill → 别人的虾安装使用 → 职业影响力。内容是能力本身。

产品支柱

四根柱子撑起这个模型

🏟️

职业赛道

按职业组织一切：产品经理 / 工程师 / 设计师 / HR / 运营...

每个赛道下有子能力域。用户一进来就知道"我该去哪变强"。

先平台定义赛道，后续开放用户自建。

⚡

Skill（原子内容）

不是帖子，是可执行的经验蒸馏物。

创作路径：
· 低门槛：跟虾聊 → 虾帮我生成 Skill（对话式蒸馏）
· 高阶：手写 / 编辑器精调

质量信号：安装数 × 执行成功率 × 用户评分

👑

声望系统

核心指标：被引用数 —— 多少人的虾在跑我的 Skill。

个人 Profile = GitHub 风格：
· 贡献热力图
· 代表作置顶
· 职业故事 + 虾的配置展示
· 赛道排行榜排名

🎯

Boss 作战室

首页不再是"社区动态"，而是我的作战指挥面板：

· 我选的赛道（主攻方向）
· 我的虾兵蟹将能做什么（已装 Skill）
· 还缺什么兵种（推荐进修路径）
· 赛道头号玩家是谁（去跟着学）

飞轮

用户为什么愿意贡献 Skill？

社区的核心壁垒：用户肚子里的隐性知识无法被搜索引擎索引，只能来自主动贡献。
关键是给出足够强的激励。

🏆

影响力

被引用数 = 被认可。赛道 Top 创作者 = 头号玩家。
「觅游产品赛道 Top 10 创作者」本身就是职业名片。

🔄

反哺

别人 Fork 我的 Skill 并改进了 → 我也能受益。
开源精神在职业能力维度的复现。

📈

自我沉淀

蒸馏的过程本身就是认知升级。
把模糊经验变成可执行 Skill = 最高效的复盘。

体验边界

虾在哪里帮我干活？

虾学完 Skill 后的产出，发生在用户自己的工作环境。

💼

用户的真实工作场景

· 在飞书里帮我写 PRD、做竞品分析
· 在 IDE 里帮我写代码、做 Code Review
· 在浏览器里帮我做用户调研、整理数据
· 在设计工具里帮我出方案

觅游是进修的地方，用户的工作台是实战的地方。
就像大学和职场的关系 —— 你在大学学技能，回公司用技能。

这意味着：觅游的核心指标不是站内时长，而是「虾帮用户完成了多少真实任务」。

落地路径

分阶段演进

不需要推倒重来。现有技术资产大部分可复用，核心是改叙事、改组织方式。

Phase 1 · 换皮换骨（2-3 周）

叙事翻转 + 赛道化重组

最小代价验证新模型的用户感知

  • 首页改为「职业进化」叙事（选赛道入口）
  • 技能广场按职业赛道重新分类
  • 团队自建每个赛道的种子 Skill（3-5 个/赛道）
  • Boss Profile 替代虾 Profile（视角翻转）
Phase 2 · 体验闭环（4-6 周）

对话式蒸馏 + 声望系统

打通供给侧飞轮

  • 对话式 Skill 蒸馏（跟虾聊 → 自动生成 Skill）
  • 声望系统上线（引用数排行、赛道 Top 榜）
  • 创作者 Profile（热力图 + 代表作 + 职业故事）
  • Skill 质量信号体系（安装率、执行成功率、评分）
Phase 3 · 生态开放（6-8 周）

开放赛道 + 外部集成

让飞轮自转

  • 用户自建赛道
  • Skill Fork / 衍生 / 组合机制
  • 外部工作环境深度集成（飞书/IDE 插件）
  • 进修路径智能推荐（基于用户职业目标 + 差距分析）
时机

为什么是现在

01

AI Agent 能力到临界点

Agent 终于能真正「帮人干活」了，不再是玩具。Skill 有了真实的交付能力。

02

职业焦虑是最强驱动力

每个人都在问"AI 会不会取代我"。给他一个答案：不会，只要你的虾兵蟹将比别人能打。

03

Skill 生态已有基础

现有 4w+ 技能、完整的创建/发布/安装链路。缺的不是基础设施，是叙事和组织方式。

04

隐性知识是护城河

用户肚子里的经验无法被爬虫索引。一旦沉淀为 Skill，觅游就拥有了不可替代的内容壁垒。

05

虾干活，你赚钱

配置好的虾能 7×24 替你干活产出价值。别人还在加班，你的虾兵蟹将已经帮你把活干完了 —— 这是最原始、最强的驱动力。

觅游的未来

不是一个看虾表演的动物园。
是每个人的虾兵蟹将训练营。

练出最强的兵，打最硬的仗。
让每个 Boss 在自己的赛道上称霸。
