# 蒸馏 Distill — 汇报演讲稿

> 预计时长：10-15 分钟
> 配合 Demo 演示：http://localhost:3000（或 Railway 部署地址）

## 开场（1 分钟）

大家好，今天跟大家分享一个我最近在思考的产品方向——**蒸馏**。

先问大家一个问题：你每天有多少工作是在重复做？写周报、整理会议纪要、做竞品分析、回复需求邮件……这些事你已经做了无数遍，形成了自己的套路，但每次还是得亲手做一遍。

**蒸馏要解决的就是这个问题**：让 AI Agent 观察你的工作方式，学会你的套路，然后主动帮你干。

一句话定位：**蒸馏是一个 AI Native 的职业能力平台。**

Slogan：蒸馏世界，成就自己。蒸馏自己，成就世界。

---

## 第一部分：为什么做这件事（2 分钟）

### 现状的问题

现在的 AI 工具有个矛盾：

- AI 越来越强了，能帮人干很多活
- 但**你得告诉它怎么干**——每次都得从头教一遍

举个例子：你让 AI 帮你写 PRD，你得告诉它格式是什么、要包含哪些模块、你们团队的规范是什么。下次再写，你还得再说一遍。

**这些「怎么做」的经验，其实就在你脑子里，但没有人帮你把它沉淀下来。**

### 市场机会

- AI Agent 的能力已经到了能真正帮人干活的临界点
- 每个人肚子里都有隐性知识——这是搜索引擎和爬虫抓不到的
- 职业焦虑是最强驱动力——「AI 会不会取代我」的答案是：不会，但前提是你得让 AI 帮你变强

## 第二部分：蒸馏是什么（3 分钟）

### 四个产品哲学

**1. 人都是懒的**

不要指望用户主动来蒸馏自己。所以我们的策略是：Agent 主动去观察、主动去蒸馏、主动来帮忙。

**2. 蒸馏物是活的**

蒸馏出来的东西不是文章、不是帖子，它是**可以被 Agent 直接执行的能力**。用户可能完全无感就享受到了好处。

**3. 分享是零成本的**

Agent 已经帮你蒸馏好了、跑验证了、写好描述了。你只需要点一下「好，分享」。

**4. 智能发生在边缘**

所有推理、匹配、打分都发生在用户自己的 Agent 里。平台只是仓库和交易市场。这意味着隐私有保障、个性化无上限、离线也能工作。

### 蒸馏物的两个层级

- **Skill**：单个方法论。比如「30 分钟搞定 PRD」、「A/B 实验设计」
- **Plugin**：完整的工作系统。Skill + Agent 人格 + 自动触发规则 + 外部集成

## 第三部分：怎么运转的（3 分钟）

### 四条核心链路

**链路 1：Agent 主动蒸馏（供给侧）**

这是最核心的创新。流程是：

1. Agent 在后台持续观察你的工作（对话历史、浏览行为、工作节奏）
2. 发现你重复做同一件事 → 自动蒸馏成 Skill
3. 先悄悄帮你跑几次，验证确实好用
4. 验证通过后才问你：「这个帮你用了 5 次了，社区有人需要，分享出去？」

关键设计：**不是蒸馏完就问你要不要公开，而是验证好用之后再问。**

**链路 2：Agent 主动找蒸馏物（消费侧）**

你让 Agent 帮你干活时，它发现社区有更好的方法，自动装上就用。你甚至不用知道。

**链路 3：定时任务 → 场景预判**

Agent 知道你每周五写周报 → 周五直接把周报初稿送到你面前。不是推通知，是活已经干完了。

**链路 4：AI 搜索 + 组合编排**

你说「我要做产品立项评审」→ Agent 组合 3 个蒸馏物 → 按顺序执行 → 产出完整立项材料。

### 分享飞轮

为什么用户会愿意分享？

- **零成本**：Agent 已经帮你做好了一切，你只需确认
- **已验证**：不是「可能有用」，是「帮我用了 N 次」
- **有人需要**：Agent 会告诉你社区有多少人在找类似的
- **声望**：分享后被引用 → 排名升

## 第五部分：技术可行性（1 分钟）

这不是空想。OpenClaw 已有的「Memory Dreaming」机制跟我们要做的自动蒸馏完全同构：

- 已有基础设施覆盖 80%（对话扫描、心跳唤醒、Skill 审批流程都是现成的）
- 需要新开发的大约 8-10 周
- **最小验证 4 周就能跑通核心循环**

第 1 周开启自动扫描 → 第 2-3 周加语义聚类 → 第 4 周做分享确认 UI → 闭环。

---

## 第六部分：竞争优势（1 分钟）

为什么别人做不了？

1. **隐性知识壁垒**：用户脑子里的方法论，爬虫抓不走，只有本地 Agent 能观察到
2. **验证过的质量**：每个蒸馏物都是跑了 N 次确认好用的，不是未验证的「攻略」
3. **可执行**：不是文章，是能直接跑的能力
4. **网络效应**：越多人用 → 越多蒸馏 → 越多好内容 → 越多人用
5. **场景绑定**：通过 Plugin 渗透到用户日常工作流，迁移成本极高

## 收尾（30 秒）

总结一下：

蒸馏的核心逻辑是——**你干活，Agent 偷偷学。学会了主动帮你干。干好了帮你分享。分享了让你更牛。**

这是一个供给侧几乎零摩擦的能力平台。不需要用户「创作」，Agent 会帮他蒸馏。不需要用户「逛广场」，Agent 会帮他找。

## 预判 Q&A

**Q：跟 Anthropic 的 Skill Marketplace 有什么区别？**
A：Anthropic 的 Marketplace 是手动创建、手动安装。我们的核心差异是 **Agent 主动蒸馏 + 验证后分享**——用户什么都不用做。而且我们有 AI 组合推荐能力，不只是单个 Skill 的安装。

**Q：用户的隐私怎么保护？**
A：所有观察和分析都发生在用户本地 Agent 里。只有用户明确确认分享的蒸馏物才上传到平台。工作数据永远不会离开用户的设备。

**Q：冷启动怎么解决？初期没有内容怎么办？**
A：两条腿走路。一是种子内容——已经有 30+ 个基于开源项目的高质量蒸馏物（来自 Anthropic、obra/superpowers、deanpeters 等）。二是自动蒸馏——只要用户装了 Plugin，Agent 就开始工作，供给侧是自动的。

**Q：跟得到、知乎这些知识平台有什么区别？**
A：得到卖的是知识（听了就行），我们卖的是能力（能帮你执行）。知乎是人写给人看的，我们是 Agent 写给 Agent 用的。本质区别是：**我们的内容是可执行的，不是可阅读的。**

**Q：商业模式是什么？**
A：初期不考虑变现，先做规模。长期看：高级蒸馏物付费、企业版（团队知识库蒸馏）、API 调用收费（Agent 从社区安装 Skill 的接口费）。

**Q：4 周验证具体怎么做？**
A：第 1 周开启 OpenClaw 的 autonomous 模式 + 心跳扫描；第 2-3 周加 embedding 聚类找重复模式；第 4 周做确认分享 UI。找 5-10 个内部用户先跑起来看数据。
