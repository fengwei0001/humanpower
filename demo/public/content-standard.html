蒸馏内容标准

得到卖知识——听完你「知道」了。蒸馏卖能力——装完你「能做到」了。

根本判断

一个东西该不该成为蒸馏物，只有一个问题：

用户的 Agent 拿到它之后，能不能直接帮用户把一件具体的事干完？

能干完 → 是蒸馏物。干不完 → 不是。

一、蒸馏物是什么

蒸馏物是可被 Agent 执行的能力单元。不是知识、不是建议、不是文章。是一段结构化的经验，Agent 拿到后能立刻开始工作，并产出一个具体的交付物。

蒸馏物是蒸馏物不是
一个动作一个概念
一次执行能完成的需要几周才能见效的
有明确输入和输出的「看情况」「因人而异」的
交付物可感知的「提升思维」「拓展视野」的
比 AI 裸跑效果好的跟直接问 AI 差不多的

二、五条硬标准（准入门槛）

不满足任何一条，就不该作为蒸馏物发布。

1. 一次执行，一个交付物

蒸馏物必须能在一次 Agent 执行中跑完，并产出一个具体的东西——一份文档、一个方案、一张表、一组数据、一段代码。

✅ 合格❌ 不合格
帮你生成一份本周周报提高团队的数据驱动能力
分析这组数据的异动原因建立良好的用户增长体系

2. 输入具体，输出明确

用户必须清楚知道：给什么进去、出来什么。不存在「看情况」。

✅ 合格❌ 不合格
输入：产品 idea → 输出：一份 PRD输入：「你的困惑」→ 输出：「一些启发」
输入：数据 + 异动描述 → 输出：分析报告输入：不确定 → 输出：不确定

3. 有方法论，不是空壳

内部必须有真实的方法论——步骤、判断逻辑、经验法则。不是简单包装了一个 prompt。

✅ 「先看留存曲线拐点在哪天，因为拐点之前是激活问题，拐点之后是价值问题」

❌ 「第一步：分析数据」（空话，没有 how）

4. 比 AI 裸跑强

如果直接跟 AI 说同样的话能得到差不多的结果，这个蒸馏物就没有存在价值。

判断方式：去掉这个蒸馏物，直接问 AI 同一个问题，结果差多少？

差很多 → 有价值（含独特方法论/行业经验/团队规范）

差不多 → 没价值（只是对 AI 能力的简单封装）

5. 场景边界清晰

必须说清楚适合什么、不适合什么。万能的 = 没用的。

✅ 合格❌ 不合格
适合 B2B SaaS 的试用转付费场景适合所有产品经理
适合 10 人以下团队的日常站会通用写作

三、三条质量标准（好不好）

1. 有观点、有立场

不是列举所有可能性让用户选，而是给出明确建议。

✅ 好❌ 差
「先做 A 再做 B，因为 B 依赖 A 的结果」「你可以做 A 也可以做 B，各有优劣」

2. 有踩坑经验

来自真实实践，包含「哪里容易出错」「我踩过什么坑」。这是 AI 自己生成不了的。

✅ 「第 3 步很多人会跳过，但如果不做，后面一定会返工」

✅ 「这里千万别用 XX 方法，看起来快但后患无穷」

3. 结果质量可验证

产出的结果有明确的质量标准——用户能判断「这次跑得好不好」。

✅ 可验证❌ 无法验证
输出的 PRD 能直接通过评审帮你打开思路
生成的实验方案，数据同学挑不出毛病给你一些灵感

四、表达标准（吸不吸引人）

名字：从痛苦出发

✅ 好名字❌ 差名字
30 分钟搞定 PRDPRD 结构化写作工具
一个人开完评审会多角色评审模拟器
数据跌了？5 分钟定位数据分析报告生成
公式：[时间/数量承诺] + [用户想完成的事]　或　[痛苦场景]？+ [解决方案]

描述：先说爽点

✅ 好描述❌ 差描述
再也不用对着空白文档发呆了。把老板那句话丢进来，出来就是能过评审的 PRD。将模糊的产品需求转化为评审就绪的 PRD，包含五章节标准文档输出。
公式：[解决什么痛苦] + [给什么进去] + [得到什么结果]

场景：画面感

✅ 有画面❌ 无画面
周一老板扔来一句话，周三要评审，你慌不慌？当你需要快速输出产品需求文档时

步骤：人话

✅ 人话❌ 术语
把你脑子里那坨混沌想法倒出来澄清阶段——提取已知信息，识别缺口

语气：骄傲地分享

像朋友说「这个方法帮了我，推荐给你」，不是说明书。

五、颗粒度判断

❌ 太大了（是问题，不是动作）

怎么提高留存 / 怎么赚钱 / 怎么不花钱增长 / 怎么做好产品 / 怎么管理团队

→ 这些应该用工具包（多个蒸馏物组合）或 AI 组合推荐来回答

❌ 太小了（AI 裸跑就能搞定）

翻译一段话 / 给变量起名 / 格式化 JSON / 改个错别字

→ 不需要额外方法论，直接问 AI 就行

✅ 刚刚好

一个会议开完 → 输出结构化纪要

一个 PRD 写完 → 从 6 个角色角度挑刺

一组数据给我 → 定位为什么指标跌了

一个功能描述给我 → 设计完整的 A/B 实验方案

判断公式：一次给输入 → 一次跑完 → 一个拿得出手的交付物
好的蒸馏物 = 一次执行干完一件具体的事 + 内含真实经验让 Agent 比裸跑强 + 名字让人一眼就知道自己需不需要

六、团队审核 Checklist

每个蒸馏物发布前，过一遍：

  • Agent 能一次执行完吗？（不能就拆或改为工具包）
  • 输入输出明确吗？（用户知道给什么、拿到什么）
  • 比直接问 AI 强吗？（有独特方法论/经验/规范）
  • 场景边界说清了吗？（适合什么、不适合什么）
  • 名字从痛苦出发了吗？（不是功能描述）
  • 描述第一句让人有感觉吗？（不是冷冰冰的说明）
  • 步骤是人话吗？（朋友教你 vs 教科书）
  • 有踩坑经验吗？（不只是正确路径，还有错误路径）
  • 粒度对吗？（不太大也不太小）
