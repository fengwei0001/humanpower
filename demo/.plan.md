# yunAgent 部署方案

## 目标
在 Railway 上部署 Hermes Agent 作为觅游的云端执行引擎，通过 API 接收觅游前端的请求，执行 Skill 并返回结果。

## 架构
- 使用 Hermes 自带的 Web Server（FastAPI + uvicorn，端口 9119）
- 利用 Hermes 的 Profile 系统做用户隔离（每个觅游用户 = 一个 Hermes Profile）
- 暴露 `/api/pty` WebSocket 给觅游前端，实现实时聊天/执行

## yunAgent 仓库内容

```
yunAgent/
├── Dockerfile          # 基于 hermes-source 的 Dockerfile，调整为 Railway 兼容
├── railway.json        # Railway 部署配置
├── docker-compose.yml  # 本地开发用
├── config/
│   └── default.yaml    # Hermes 默认配置（模型、工具集等）
└── README.md
```

## 关键配置
1. Railway 环境变量：
   - `PORT` = Railway 分配的端口
   - `OPENROUTER_API_KEY` 或 `OPENAI_API_KEY` — LLM 后端
   - `API_SERVER_KEY` — API 鉴权密钥（觅游后端调用时带上）

2. Hermes 启动命令：`hermes web --host 0.0.0.0 --port $PORT --allow-public`

3. 觅游前端通过 WebSocket 连接 `/api/pty` 实现实时对话

## 步骤
1. 创建精简 Dockerfile（基于 hermes-source，去掉 s6-overlay 复杂度，直接 CMD 启动 web server）
2. 创建 railway.json 配置
3. 推送到 GitHub，Railway 自动部署
4. 验证 API 可访问
