# 技能广场改版：大卡片 + 场景化筛选

## 目标
把现在的 Square 页面从"热门列表 + 侧边栏"改成截图中的大卡片 2 列网格布局，
配合更好的场景化筛选，让技能"更全、更好找"。

## 设计方案

### 1. 页面布局
- **标题区**：保留"技能广场" + 副标题
- **筛选区**：赛道 tabs + 标签筛选（场景化分类）
- **卡片网格**：2 列大卡片，样式对标截图
- **去掉右侧边栏**：信息整合到卡片本身
- **加载更多**：底部加载按钮

### 2. 大卡片样式（对标截图）
每张卡片包含：
```
┌─────────────────────────────────────────┐
│ **AI会主动帮你干活**                       │  ← 标题（bold, 大字）
│ 再也不用催着AI干活了。装上这个...           │  ← 描述（灰色，2行截断）
│                                          │
│ 🎯产品经理   featured   ai-lab           │  ← 赛道 tag + 分类 tags
│                                          │
│ 🔥169,782  🔗0  ⭐0.00    🧑‍💻 halthelobster │  ← 安装数 + 引用 + 评分 + 作者
└─────────────────────────────────────────┘
```

### 3. 场景化筛选（不只是赛道）
- **第一层**：赛道 tabs（产品经理 / 工程师 / 自媒体 / OPC）+ 全部
- **第二层**：场景标签（根据 `tags` 和 `sub_domain` 动态聚合）
  - 产品经理下：PRD、数据分析、用户研究、汇报演示...
  - 工程师下：tools、前端开发、Code Review、自动化...
  - 自媒体下：视频创作、小红书、文案写作...
  - OPC：产品构建、增长获客、自动化...
- **排序**：热门 / 最新 / 评分

### 4. 后端改动
- **新增 API**: `/api/skills/tags?track=pm` — 返回该赛道下高频 tag 列表
  （从 `tags` 字段聚合 top 15 高频标签）
- **skills API 支持 tag 筛选**：加 `tag` 参数

### 5. 前端改动
- **重写 Square.tsx** — 大卡片网格 + 筛选
- **复用 SkillCard 组件** — 或创建 SkillCardLarge 组件（对标截图大卡片样式）
- **新增筛选 hooks** — 赛道 + tag 联动

### 6. 去掉"成功率"展示
- 卡片底部只展示：安装数、引用数、评分
- 不再展示 `successRate`（因为是假数据）

## 文件变更清单
1. `demo/server.cjs` — 新增 `/api/skills/tags` 端点 + skills API 支持 tag 参数
2. `demo/src/pages/Square.tsx` — 完全重写
3. `demo/src/components/SkillCard.tsx` — 微调或创建 SkillCardLarge（匹配截图样式）
4. `demo/src/services/skills-api.ts` — 新增 fetchTags 方法 + fetchSkills 支持 tag 参数
