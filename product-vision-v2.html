蒸馏是一个 AI Native 的职业能力平台。用户的本地 Agent 观察工作 → 自动蒸馏经验为可执行的能力包 → 帮用户干活 → 分享到社区帮助更多人。
一

产品哲学

😴

人都是懒的

主动蒸馏自己的人永远是少数。Agent 观察用户 → 主动发现模式 → 自己蒸馏 → 拿给用户确认。

⚡

蒸馏物是活的

不是文章，不是用来看的。能被 Agent 直接执行，用户可能无感就享受到了好处。效果可度量。

🎁

分享是零成本的

Agent 帮你蒸馏好了、跑验证了、写好描述了。你只需要点一下「好，分享」。

🧠

智能发生在边缘

所有推理、匹配、打分都发生在用户自己的 Agent 里。平台是仓库，智能跑在离用户最近的地方。

「智能在边缘」的意义：

🔒 隐私天然有保障 — 工作数据不需要上传

🎯 个性化无上限 — 每个 Agent 基于自己主人的上下文决策

📴 离线也能工作 — 蒸馏物装到本地后不依赖平台

♾️ 平台不是瓶颈 — 10 万用户的 Agent 各自在本地跑

二

核心概念：蒸馏物的两个层级

蒸馏产出的东西统一称为蒸馏物。它有两个层级：

层级是什么蒸馏什么类比
Skill单个可执行的经验方法论一个能力点一首歌
Plugin完整的工作系统方法论 + Agent 人格 + 自动化 + 集成一整套装备

Skill — 经验的最小蒸馏单元

回答「怎么做某件事」—— 怎么写 PRD、怎么做竞品分析、怎么设计实验。一段结构化的、可被 Agent 执行的经验。

Plugin — 工作系统的完整蒸馏

真实工作中，一个能力点往往不够：

📝 写完 PRD 自动触发评审（事件钩子）

🧠 一个懂 PM 思维的 Agent 人格帮你把关（专属 Agent）

👁️ 后台观察你的工作模式持续优化建议（后台监控）

🔌 连接飞书/GitHub 把产出直接送到该去的地方（外部集成）

🧩 好几个 Skill 组合在一起完成一个完整任务（能力组合）

这些加在一起就是 Plugin ——「工作系统蒸馏物」。

Plugin 的构成

组件作用举例
skills/多个 Skill 组合一个工具包解决一类完整问题
agents/自定义 Agent 人格带 PM 思维的专属助手
hooks/事件钩子保存文件后自动触发下一步
monitors/后台监控持续观察，发现新蒸馏机会
mcp外部服务连接飞书/GitHub/Slack 集成
bin/可执行工具命令行工具
settings默认配置一键切换工作模式

四

四条核心链路

🔄 链路 1：Agent 主动蒸馏（供给侧）
  • Agent 持续观察用户工作（对话 / 浏览 / 工作节奏）
  • 发现重复模式 → 自主蒸馏为 Skill 或 Plugin 草稿
  • 先自己拿来帮用户干活（悄悄验证）
  • 跑了 N 次、用户没改 → 确认好用
  • Agent：「帮你用了 8 次了，社区有人需要，分享出去？」
  • 用户确认 → 发布平台 → 声望 ↑
⭐ 不是蒸馏完就问「要不要公开」，而是验证好用之后再问。
🔍 链路 2：Agent 主动找蒸馏物（消费侧）
  • Agent 帮用户干活时意识到「有更好的方法」
  • 实时查询蒸馏平台 API
  • 平台返回匹配的蒸馏物（或组合方案）
  • Agent 直接装上执行 — 用户可以无感
⏰ 链路 3：定时任务 → 场景预判 → 直推本地 Agent
  • 定时任务触发（周五 14:00）
  • Agent 知道「每周五写周报」→ 拉取 Skill + 预生成草稿
  • 直接说：「周报写好了，你看看改改？」
⭐ 不是推通知，不是发邮件，是活已经干完了直接送到面前。通过本地 Agent 心跳直达。
🧩 链路 4：AI 搜索 + 组合编排（问题驱动）
  • 用户：「下周要做新产品立项评审」
  • Agent 查询平台 → 返回组合：① 写新闻稿 ② 算市场规模 ③ 做演示
  • Agent：「按这顺序走一遍？」→ 用户：「好」→ 自动依次执行

六

内容组织：赛道 + 工具包

职业赛道（官方视角）

按职业组织：产品经理 / 工程师 / 设计师 / 运营 / HR / ... 每个赛道有子能力域。一个蒸馏物可以同时属于多个赛道。

工具包（用户视角）

像网易云工具包一样，蒸馏物可自由组合为收藏集：

类型示例
新手入门包「新手 PM 必备 10 个 Skill」
场景组合包「从 0 到 1 做一次完整的产品立项」
大佬配置单「Top 1 PM 的虾兵装备清单」
公司/团队包「XX 团队新人 Onboarding 必装」
AI 智能推荐Agent 基于能力缺口自动生成

互动：一键安装 / Fork / 订阅更新 / 引用计入声望。

渠道方式发现什么

十一

竞争优势

壁垒解释
隐性知识只有本地 Agent 能观察到，爬虫抓不走
验证过的质量每个蒸馏物都跑过 N 次确认好用
可执行不是文章，是能直接跑的能力
网络效应越多人用 → 越多蒸馏 → 越多好内容
场景绑定渗透工作流，迁移成本极高
边缘智能不依赖平台算力，天然可扩展
